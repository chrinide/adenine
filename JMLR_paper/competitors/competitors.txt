1) Divvy (published on JMLR MLOSS):
	- PRO: interactive GUI, parallel
	- CONS: working only on OSX, awful plots, Xcode project (C-ish), few algorithms (K-means, hierarchical + PCA, Isomap, t-SNE), cannot handle missing values

2) GGobi (published book ELSEVIER):
	- PRO: interactive GUI, R integration, longitudinal data, missing values
	- CONS: only data visualization (univariate or PCA), no clustering

3) Orange:
	- PRO: it's complete, beautiful, open source, works on python, integrated in anaconda, 	it has a biodata plugin
	- CONS: it lacks some algorithms for both DR and CL that we have (e.g.: LLE, SE, t-SNE, MS, AP, Spectral, ...)
	http://jmlr.org/papers/volume14/demsar13a/demsar13a.pdf
